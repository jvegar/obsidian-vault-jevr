# 2026-03-05_Monorepo-vs-Multirepo

## Mono-repositories
- Repositories that contain multiple projects
### Benefits:
    - Allows easier sharing of code and resources between projects
    - It can save time and reduce code duplication, as developers can easily reference and reuse existing code.
    - Make it easier to manage dependencies and versioning, as everything is located in one central location.
    - It can help improve communication between teams. With all projects housed in one repository, it's easier fo differents teams to work together and share their work.
    - It can help to ensure consistency across projects, as all code is located in one place and can be easily reviewed and approved.
### Drawbacks:
    - It can become quite large and complex over time, which can make difficult to mantain and manage.
    - It can be challenging to ensure that all code changes are properly tested and reviewed.
